# LLVM Lit + ttrt run Testing Pipeline

This document describes how MLIR lit tests and `ttrt run` work together to
compile and execute tests on Tenstorrent silicon hardware in CI.

## Overview

Each `.mlir` test file under `test/ttmlir/Silicon/` contains `RUN` lines that
form a three-stage compilation-and-execution pipeline:

```mlir
// RUN: ttmlir-opt --ttir-to-ttnn-backend-pipeline="system-desc-path=%system_desc_path%" -o %t.mlir %s
// RUN: ttmlir-translate --ttnn-to-flatbuffer -o %t.ttnn %t.mlir
// RUN: ttrt run %t.ttnn
```

1. **`ttmlir-opt`** -- lowers MLIR through the compiler pipeline (TTIR -> TTNN).
2. **`ttmlir-translate`** -- serializes the lowered MLIR to a flatbuffer binary (`.ttnn`).
3. **`ttrt run`** -- executes the flatbuffer on actual silicon hardware.

LLVM lit orchestrates all three steps as a shell test. If any step fails, the
test fails.

## Test Directory Structure

```
test/ttmlir/Silicon/
├── TTNN/
│   ├── n150/              # Single-chip tests (eltwise, matmul, reduction, optimizer)
│   ├── n300/              # Two-chip tests
│   └── llmbox/
├── TTMetal/
│   └── n150/              # TTMetal backend tests
└── StableHLO/
    └── n300/              # StableHLO frontend -> silicon tests
```

Tests under `Silicon/` are the ones that actually run on hardware via `ttrt
run`. Non-silicon tests (under `test/ttmlir/Dialect/`, `test/ttmlir/Conversion/`,
etc.) only test compilation, not execution.

## Lit Configuration

The lit config at `test/lit.cfg.py` sets up:

- **Feature flags**: `stablehlo`, `opmodel`, `perf` -- tests use `// REQUIRES:`
  to gate on these.
- **Substitutions**: `%system_desc_path%` resolves to the path to the system
  descriptor file.
- **Tool paths**: `ttmlir-opt`, `ttmlir-translate`, `ttrt`.
- **Parallelism groups**: opmodel tests run single-threaded to avoid device
  contention.

## How CI Runs These Tests

The flow in `.github/workflows/call-test.yml`:

1. **`ttrt query --save-artifacts`** -- generates a system descriptor
   (`system_desc.ttsys`) that describes the target hardware.
2. **`llvm-lit`** -- runs the `.mlir` test files, which internally invoke
   `ttmlir-opt` -> `ttmlir-translate` -> `ttrt run`.
3. **Test scripts** (`.github/test_scripts/`) handle specialized cases:
   - `ttrt.sh` -- general ttrt silicon tests.
   - `op_model_ttrt.sh` -- performance model tests (runs lit first, then
     `ttrt run` separately on generated flatbuffers with benchmarking).

### Test Matrix

The test matrix is generated by `.github/scripts/python/generate_test_matrix.py`,
which maps tests to the correct CI runners (n150, n300, llmbox machines). The
matrix is consumed by `.github/scripts/python/run_tests_from_json.py` which
executes each test script with the appropriate environment variables.

### Key Environment Variables

| Variable | Description |
|---|---|
| `SYSTEM_DESC_PATH` | Path to the system descriptor (e.g., `ttrt-artifacts/system_desc.ttsys`) |
| `BUILD_DIR` | CMake build directory |
| `TTRT_REPORT_PATH` | Output path for ttrt JSON results |
| `TEST_REPORT_PATH` | Output path for XML test report |
| `RUNS_ON` | Target machine type |

## Performance Tests

Performance tests live under
`test/ttmlir/Silicon/TTNN/n150/optimizer/models_perf_tests/` and use
`--benchmark` mode:

```mlir
// REQUIRES: opmodel, perf
// RUN: ttmlir-opt --ttir-to-ttnn-backend-pipeline="system-desc-path=%system_desc_path% optimization-level=1" -o %t.mlir %s
// RUN: ttmlir-translate --ttnn-to-flatbuffer -o %t.ttnn %t.mlir
// RUN: ttrt run --benchmark %t.ttnn
```

These have a dedicated CMake target:

```bash
cmake --build build --target check-perf
```

## Running Tests Locally

```bash
# Activate the environment
source env/activate

# Generate a system descriptor (requires hardware)
ttrt query --save-artifacts
export SYSTEM_DESC_PATH=$(pwd)/ttrt-artifacts/system_desc.ttsys

# Run all compiler tests (includes silicon tests if on a machine with hardware)
cmake --build build --target check-ttmlir

# Run a specific lit test
llvm-lit test/ttmlir/Silicon/TTNN/n150/eltwise/add.mlir

# Run perf tests
cmake --build build --target check-perf
```

## ttrt Tool

`ttrt` is the runtime tool located at `tools/ttrt/`. Key subcommands:

| Subcommand | Description |
|---|---|
| `ttrt run` | Execute a flatbuffer on device |
| `ttrt run --benchmark` | Execute with benchmarking enabled |
| `ttrt query --save-artifacts` | Generate a system descriptor |
| `ttrt read` | Inspect flatbuffer binary sections |
| `ttrt perf` | Performance profiling |
| `ttrt check` | Validation |

### Common `ttrt run` Arguments

- `--clean-artifacts` -- clean artifacts from previous runs.
- `--save-artifacts` -- save artifacts during run.
- `--program-index` -- which program to run (default: `all`).
- `--loops` -- number of execution loops.
- `--init` -- tensor initialization method (e.g., `randn`).
- `--benchmark` -- enable benchmarking mode.
- `--rtol`, `--atol` -- tolerance for golden comparison tests.
